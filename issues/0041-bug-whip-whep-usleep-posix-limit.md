# whip/whep の usleep を POSIX 上限内に収める

- Priority: Medium
- Polished: 2026-06-05
- Created: 2026-06-05
- Model: Opus 4.8

## 目的

`webrtc/src/whip.c` と `webrtc/src/whep.c` で、`usleep` に 100 万マイクロ秒以上の値を渡している。
POSIX の `usleep` は引数が 100 万マイクロ秒未満であることを要求しており、これを超える呼び出しは未定義動作となる。
適切な API へ置き換え、規定の範囲内でスリープするように修正する。

## 優先度根拠

未定義動作であり、環境によっては期待どおりに待機しない、あるいはエラーになる可能性がある。
ただし該当箇所はサンプル/動作確認用の待機であり、本番の制御フローではないため、優先度は Medium とする。

## 現状

`webrtc/src/whip.c:1594` では、10 秒の待機を意図して `usleep(10000000)` を呼んでいる。

```c
  usleep(10000000);  // 10 second
```

`webrtc/src/whep.c:1352` では、30 秒の待機を意図して `usleep(30000000)` を呼んでいる。

```c
  usleep(30000000);
```

`usleep` の引数は POSIX 上 100 万マイクロ秒未満でなければならないが、`10000000`（1000 万）も `30000000`（3000 万）もこの上限を超えており、未定義動作となる。

## 設計方針

- 秒単位の待機には `sleep` を用いる、あるいは `nanosleep` で待機時間を指定するなど、長時間の待機を規定どおり扱える API に置き換える。
- 置き換え後も、それぞれ 10 秒・30 秒という意図した待機時間を維持する。
- いずれの呼び出しも POSIX の定める引数範囲内に収める。

## 完了条件

- `webrtc/src/whip.c` と `webrtc/src/whep.c` のスリープ呼び出しが、POSIX の定める引数範囲内に収まっている。
- 待機時間が従来の意図（10 秒・30 秒）と一致している。
